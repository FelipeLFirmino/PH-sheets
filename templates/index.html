1. Tipos de Tributação de ICMS

Cada produto em uma NF-e pode ter um regime tributário diferente. O sistema identifica automaticamente o regime de cada item:

RegimeCST TípicoO que significa
Normal00Tributação padrão. Cada elo da cadeia paga seu próprio ICMS na venda.
ST10 / 60Substituição Tributária — a indústria ou importador já recolheu o ICMS de toda a cadeia com base em um preço presumido.
ANT—Antecipação Tributária — ocorre quando não há acordo de ST com o estado de origem. O destinatário (você) recolhe ao receber a mercadoria.
Isento40 / 41Produto isento de ICMS. Nenhum imposto é devido na cadeia.
Simples102 / 500Fornecedor no Simples Nacional — regime simplificado sem transferência de crédito de ICMS.
O regime é identificado pelo campo CST no XML da NFe, combinado com os valores retornados pela API SEFAZ AL.
2. Como o Sistema Identifica o Regime de Cada Produto

A classificação segue esta ordem de prioridade:

  1. ST — se ST unitário > R$ 0,005 (valor vindo do XML campo vICMSST + API SEFAZ)
  2. ANT — se ANT unitário > R$ 0,005 e ST = 0 (valor exclusivo da API SEFAZ, tipo 'ANT')
  3. Isento — se CST for 40, 41, 50, 60, 102 ou 500
  4. Normal — qualquer produto que não se enquadre nos anteriores
Threshold de segurança: O sistema usa > 0,005 (e não > 0) como mínimo para evitar falsos positivos causados por arredondamento de ponto flutuante. Valores como 4.5e-13 retornados pela API não ativam o regime ST/ANT.
3. Crédito de ICMS — Quando se Aplica

O crédito de ICMS é o valor que você pode abater do ICMS que vai pagar na venda. O ICMS é não-cumulativo: você desconta o que pagou na compra do que deve na venda.

Quando o crédito é ZERO
CondiçãoMotivo
Produto com ST > 0O ICMS já foi pago por toda a cadeia pelo substituto. Sua venda sai sem débito de ICMS — então não há crédito a abater.
CST 40, 41, 50 (isento/suspenso)Sem débito na saída → sem crédito na entrada.
CST 60 (ST anterior)Mesmo caso de ST acima.
CST 102 / 500 (Simples Nacional)Regime simplificado — o fornecedor não transfere crédito de ICMS.
Quando o crédito É MANTIDO
CondiçãoMotivo
Produto com ANT > 0 (antecipação padrão)ANT padrão NÃO elimina o crédito. Conforme SEFAZ AL e Lei 6.474/2004: "A antecipação não encerra a fase de tributação." Você continua no débito/crédito normal.
CST 00 (normal)Tributação padrão — crédito integral do ICMS destacado na NF.
CST 10Tributado com ST nas etapas seguintes — o fornecedor ainda destaca ICMS, gerando seu crédito.
Exceção — ANT com encerramento: Alguns produtos específicos têm ANT com encerramento de tributação (ex: calçados em AL — Art. 591-H do RICMS/AL). Nesse caso, funciona como ST e elimina o crédito. O sistema não distingue automaticamente esse caso — se você trabalha com esses produtos, o crédito deve ser zerado manualmente (setar Crédito ICMS = 0% no formulário).

4. Fluxo de Cálculo — Varejo (ordem obrigatória)

Os passos abaixo são executados nesta sequência exata. Alterar a ordem muda o resultado.

1
Custo Real = NF Unitário × Multiplicador Varejo
Ex: R$ 7,69 × 2,0 = R$ 15,38
2
Frete = Custo Real × % Frete
Ex: R$ 15,38 × 10% = R$ 1,54
3
Despesa Operacional = Custo Real × % Despesa
Ex: R$ 15,38 × 10% = R$ 1,54
4
Crédito ICMS = NF Unitário × % Crédito (ou 0 se ST/Isento)
Ex: R$ 7,69 × 4% = R$ 0,31 — será subtraído do custo
5
Custo de Entrada = Custo Real + ST + ANT + IPI + Frete + Despesa − Crédito ICMS
Todos os custos da compra, menos o crédito fiscal
6
Preço Varejo = Preço Atual do sistema (se > 0), senão Arredondar99(Custo Real × 2)
Usa o preço cadastrado no CSV. Se não houver, calcula automaticamente com margem embutida
7
Federal = Preço Varejo × % Imposto Federal (ex: 9,13%)
8
Cartão = Preço Varejo × % Taxa Cartão (ex: 4%)
9
ICMS Saída = Preço Varejo × % ICMS Venda (ex: 20%)
10
Custo de Saída = Custo de Entrada + Federal + Cartão + ICMS Saída
Tudo que você paga para comprar + tudo que paga para vender
11
Meta % = 15% padrão — editável por produto na planilha
12
Preço Mínimo Viável = Custo de Saída ÷ (1 − Meta%)
Menor preço que ainda atinge a margem mínima
13
Margem Real = (Preço Varejo − Custo de Saída) ÷ Preço Varejo
Percentual de lucro real sobre o preço de venda
Arredondar99: Função aplicada quando não há preço no sistema. Arredonda para o centavo .99 mais próximo abaixo do valor dobrado do custo.
Ex: custo R$ 10,00 → dobro R$ 20,00 → Arredondar99 → R$ 19,99
5. Fluxo de Cálculo — Atacado

O atacado usa os mesmos dados de entrada mas com parâmetros próprios. O Custo de Entrada é compartilhado com o varejo — a diferença está nos tributos de saída e no preço de venda.

14
NF ATC = NF Unitário × Multiplicador Atacado
Ex: R$ 7,69 × 1,3 = R$ 9,99 (multiplicador menor que varejo)
15
Preço ATC = Preço Varejo × (1 − Desconto Atacado)
Ex: R$ 19,99 × (1 − 15%) = R$ 16,99
16
Federal ATC = NF ATC × % Federal
Base: custo atacado (diferente do varejo que usa preço de venda)
17
Cartão ATC = Preço ATC × % Cartão
18
ICMS ATC = NF ATC × % ICMS Venda
19
Custo Saída ATC = Custo de Entrada + Federal ATC + Cartão ATC + ICMS ATC
Usa o mesmo Custo de Entrada do varejo (passo 5)
20
Margem ATC = (Preço ATC − Custo Saída ATC) ÷ Preço ATC
6. Busca de Preço no Sistema (Fallback)

O sistema tenta encontrar o preço do produto no CSV em três tentativas:

  1. Tentativa 1 — EAN (código de barras): Cruza o campo cEAN do XML com a coluna BARRA do CSV. Se encontrar preço > R$ 0,01, usa esse valor.
  2. Tentativa 2 — Referência: Se o preço não foi encontrado pelo EAN (ou era R$ 0,01), tenta pelo código interno do produto (cProd no XML × REFERÊNCIA no CSV).
  3. Tentativa 3 — Sem preço: Se nenhuma das tentativas funcionar, o produto fica como "Sem Preço" e o Preço Varejo é calculado automaticamente via Arredondar99(Custo Real × 2). Esses produtos são listados nos alertas do dashboard.
7. Consulta à API SEFAZ AL

O sistema consulta automaticamente a API da SEFAZ Alagoas para obter os valores de ST e ANT de cada produto da nota. Isso é feito pela chave da NFe (44 dígitos).

Campo retornadoUso
tipoImposto = 'ST'Somado ao valor ST do próprio XML (vICMSST)
tipoImposto = 'ANT'Usado como ANT unitário — não existe campo ANT no XML
valorIcmsCalculado + valorFecoepCalculadoValor total por item (ICMS + Fundo de Combate à Pobreza)
A API tem timeout de 10 segundos. Se estiver fora do ar, o processamento continua normalmente usando apenas os dados do XML. Os valores de ANT e ST externo ficam zerados — verifique os valores se a API falhou.
8. Detecção de Embalagem

Se a descrição do produto no XML indicar uma embalagem (ex: "GRAMPO CAIXA COM 24"), o sistema multiplica o preço unitário do sistema pela quantidade da embalagem para calcular o preço correto.

Padrões reconhecidos: CAIXA COM N, PACOTE COM N, KIT COM N, PCT C/N, CX C/N, C/N

Proteção anti-absurdo: Se o resultado da multiplicação for mais de 3× o custo esperado, o sistema assume que o cadastro já tem o preço da caixa completa e não multiplica. Isso evita erros com produtos como grampos com 5000 unidades.

11. Regras Invariantes — Nunca Devem Ser Quebradas

Estas são as restrições fundamentais do sistema. Qualquer alteração de regra deve verificar se estas invariantes continuam sendo respeitadas:

  1. ANT não zera o crédito de ICMS — ANT padrão é diferente de ST. O crédito do ICMS da NF de compra continua existindo normalmente para produtos ANT.
  2. Threshold > 0,005 em todas as comparações de imposto — nunca usar > 0 diretamente em valores de ST/ANT.
  3. A API SEFAZ nunca pode travar o processamento — está sempre em try/catch com fallback para zero.
  4. Preço Varejo nunca pode ser zero — se não há preço no sistema, Arredondar99 é aplicado automaticamente.
  5. Multiplicador é aplicado ANTES de qualquer percentual — Custo Real = NF Unit × MULT é sempre o primeiro passo.
  6. Crédito ICMS é subtraído (não somado) no Custo de Entrada — representa benefício fiscal que reduz o custo.
  7. Federal e ICMS Saída do varejo incidem sobre o Preço de Venda — são impostos da venda, não da compra.
  8. Federal ATC e ICMS ATC incidem sobre NF ATC (custo atacado) — base diferente do varejo.
  9. O Custo de Entrada é compartilhado entre varejo e atacado — a entrada é a mesma, só a saída muda.
  10. ST = pêssego, ANT = âmbar — as cores das linhas identificam o regime e nunca devem ser trocadas.
